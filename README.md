# Операционная аналитика

Локальное браузерное приложение для сравнения операционных показателей между Excel-периодами.

Инструмент помогает загрузить несколько выгрузок или один файл с периодами внутри, сопоставить строки по выбранной колонке, выбрать нужные показатели, рассчитать динамику, посмотреть дашборд и выгрузить отчет. Проект рассчитан на внутреннюю операционную аналитику: KPI, качество, ошибки, скорость обработки, AHT, CSAT, продажи и другие регулярные метрики.

## Главное

- Работает полностью локально в браузере.
- Не требует backend, базы данных, облака или API.
- Не отправляет Excel-файлы и данные в интернет.
- Не привязан к конкретным названиям столбцов.
- Пользователь сам выбирает объект сравнения и показатели для анализа.
- Поддерживает режимы `Несколько файлов` и `Один файл`.
- Поддерживает сравнение двух и более периодов, включая периоды внутри одного файла.
- Поддерживает итоговый, последовательный и ручной режим сравнения.
- Поддерживает CSV/Excel-экспорт, включая PNG-графики в Excel-отчете.
- Позволяет сохранять историю анализов в `localStorage`.

## Как запустить

Откройте файл `index.html` в браузере.

Установка зависимостей не нужна: используемые библиотеки уже лежат локально в папке `vendor/`.

Поддерживаемые входные форматы:

- `.xlsx`
- `.xls`
- `.csv`
- `.tsv`

## Основной сценарий

1. Выбрать источник данных: `Несколько файлов` или `Один файл`.
2. Загрузить файлы периодов или один файл с колонкой периода.
3. При необходимости добавить дополнительные периоды через кнопку `+ Период`.
4. Выбрать формат сравнения: итоговый, последовательный или ручной.
5. В режиме `Один файл` выбрать колонку периода.
6. Проверить `Preview таблиц` на основном экране.
7. Выбрать объект сравнения.
8. Добавить один или несколько показателей.
9. Нажать `Сравнить`.
10. Изучить `Dashboard`, график, лучшие и худшие изменения.
11. Проверить `Таблицу результатов`.
12. При необходимости задать фильтры и нажать `Отфильтровать`.
13. Скачать CSV/Excel или сохранить анализ в историю.

## Интерфейс

Приложение построено как dashboard с боковой панелью и основным рабочим экраном.

В sidebar находятся:

- быстрая навигация по разделам;
- переключатель светлой и темной темы;
- выбор источника данных;
- загрузка файлов периодов или одного файла;
- выбор формата сравнения;
- ручной выбор пар периодов для сравнения;
- сопоставление колонок;
- запуск расчета и индикатор проверок;
- глобальные фильтры;
- экспорт CSV/Excel;
- сохранение анализа.

Основной экран содержит четыре раздела:

- `Preview таблиц` — первые строки и найденные заголовки;
- `Dashboard` — KPI, график и лучшие/худшие изменения;
- `Таблица результатов` — детальные строки сравнения;
- `История анализов` — сохраненные результаты.

Ширину sidebar можно менять перетаскиванием правой границы панели. В каждом основном разделе есть кнопка `?` с краткой подсказкой.

## Форматы сравнения

Перед расчетом пользователь выбирает, откуда брать периоды:

- `Несколько файлов` — каждый период загружается отдельным Excel/CSV/TSV-файлом.
- `Один файл` — все периоды находятся в одном файле, а пользователь выбирает колонку с датой, месяцем, неделей, годом или другим периодом.

В режиме `Один файл` приложение не сохраняет исходный файл и не дублирует его содержимое в истории. Для расчета оно собирает виртуальные периоды из строк выбранной колонки периода.

### Итоговый

Сравнивает последний период с первым:

```text
Период N - Период 1
```

Подходит, если нужно понять общий итог за несколько периодов.

### Последовательный

Сравнивает каждый следующий период с предыдущим:

```text
Период 2 - Период 1
Период 3 - Период 2
Период 4 - Период 3
```

Подходит для анализа динамики по неделям, месяцам или любым регулярным срезам.

### Ручной

Позволяет пользователю самому выбрать пары периодов:

```text
Февраль - Январь
Июнь - Март
2026 - 2025
```

Подходит, если нужно сравнить произвольные периоды: месяц к месяцу, год к году, квартал к кварталу или любой нестандартный срез.

В ручном режиме можно добавить несколько пар через кнопку `+ Пара`. Dashboard, график, таблица результатов и экспорт будут строиться по выбранным парам.

## Объект сравнения

Объект сравнения — это ключ для сопоставления строк между файлами.

Им может быть:

- логин сотрудника;
- ФИО;
- табельный номер;
- команда;
- офис;
- направление;
- любой другой стабильный идентификатор строки.

В режиме `Несколько файлов` важно, чтобы выбранная колонка присутствовала во всех загруженных периодах. В режиме `Один файл` объект выбирается один раз, а затем используется для всех виртуальных периодов внутри файла.

## Показатели

Показатель — это колонка, которую нужно сравнить между периодами.

Например:

- качество;
- скорость;
- ошибки;
- AHT;
- CSAT;
- продажи;
- выполнение KPI.

В режиме `Несколько файлов` для каждого показателя пользователь выбирает соответствующую колонку в каждом периоде. Если в разных периодах выбраны разные названия показателей, приложение показывает предупреждение, чтобы снизить риск ошибочного сравнения.

В режиме `Один файл` показатель выбирается один раз, потому что все периоды собраны из одной таблицы и используют одинаковую структуру колонок.

## Возможности анализа

- расчет абсолютного изменения;
- расчет процентного изменения;
- автоматическое распознавание процентных показателей;

- поиск роста, падения и отсутствия изменений;
- вывод лучших и худших изменений;
- проверка отсутствующих объектов сравнения;
- проверка дублей;
- предупреждения о различающейся структуре файлов;
- предупреждения о нечисловых значениях;
- подсветка проблем качества данных.

## Dashboard

Дашборд показывает:

- количество периодов;
- количество объектов сравнения;
- количество полных строк;
- количество найденных проблем;
- горизонтальный график динамики выбранного показателя;
- лучшие и худшие изменения по показателям.

График построен на Chart.js. Tooltip графика следует за курсором внутри активной полосы и показывает объект сравнения и динамику. Для процентных показателей динамика отображается в процентных пунктах.

## Фильтры

После расчета в sidebar становятся доступны глобальные фильтры:

- все изменения / только улучшения / только ухудшения;
- диапазон изменения `от` и `до`;
- поиск по сотруднику или объекту;
- поиск по подразделению или любому значению строки;
- применение фильтров через кнопку `Отфильтровать`;
- сброс фильтров.

Фильтры не применяются автоматически при вводе текста. Пользователь сначала задает условия, затем нажимает `Отфильтровать`. Это защищает dashboard от пересчета после каждого введенного символа.

Фильтры применяются к данным, которые отображаются в dashboard, графике и таблице результатов. Исходный расчет при этом не меняется.

## История анализов

После успешного сравнения анализ можно сохранить в историю.

Сохраняются:

- дата и время анализа;
- название анализа;
- выбранные показатели;
- выбранный объект сравнения;
- результаты сравнения;
- агрегированные данные dashboard;
- настройки, необходимые для восстановления результата.

Не сохраняются:

- исходные Excel-файлы;
- бинарные данные файлов;
- содержимое файлов целиком.

История хранится локально в `localStorage` и остается доступной после перезагрузки страницы. Максимум хранится 50 последних анализов.

В истории доступны:

- поиск по названию анализа;
- открытие сохраненного анализа;
- переименование;
- закрепление избранных анализов;
- удаление отдельной записи;
- полная очистка истории.

Если на странице есть несохраненный результат и пользователь обновляет или закрывает вкладку, приложение предупреждает о несохраненном анализе.

## Экспорт

### CSV

Экспортирует плоскую таблицу результатов.

### Excel

Формирует `.xlsx`-отчет с несколькими листами:

- `Обзор` — краткая сводка по анализу;
- `Сводка` — агрегированные показатели;
- `Сравнение` — детальная таблица сравнения;
- `Графики` — PNG-графики динамики по всем выбранным показателям;
- `Отсутствующие` — объекты сравнения, которые есть не во всех периодах;
- `Дубли` — повторяющиеся объекты сравнения и номера строк, где они найдены.

Excel-отчет формируется локально через ExcelJS. Для процентных показателей применяются Excel-форматы, чтобы значения не превращались в некорректные проценты при ручном форматировании.

## Назначение модулей

- `main.js` — связывает интерфейс, состояние приложения и бизнес-логику. Обрабатывает режимы источника данных, загрузку файлов, выбор колонок, ручные пары сравнения, запуск анализа, фильтры, историю и экспорт.
- `src/appState.js` — создает начальное состояние: режим источника, периоды, один файл, выбранные показатели, режим сравнения, ручные пары, фильтры, результаты и сообщения.
- `src/normalizers.js` — нормализует значения: пробелы, регистр, числа, проценты, пустые значения и форматирование вывода.
- `src/excelReader.js` — читает Excel/CSV/TSV через SheetJS, определяет строку заголовков, собирает preview и первичные предупреждения по структуре.
- `src/periodBuilder.js` — собирает виртуальные периоды из одного файла по выбранной колонке периода.
- `src/comparator.js` — сопоставляет строки по объекту сравнения, строит или принимает пары периодов, считает динамику, процент изменения, пропуски, дубли и нечисловые значения.
- `src/analytics.js` — готовит агрегаты для dashboard: количество объектов, проблем, улучшений, падений, лучшие и худшие изменения.
- `src/exporters.js` — формирует CSV и Excel-отчеты. Для Excel создает PNG-графики на canvas и вставляет их в отчет.
- `src/charts.js` — рисует график динамики через Chart.js и настраивает tooltip, цвета, анимацию и форматирование значений.
- `src/ui.js` — рендерит preview, выбор колонок, проверки качества данных, KPI-карточки, таблицу результатов, историю и блоки dashboard.
- `src/historyStore.js` — сохраняет, загружает, сортирует, закрепляет и очищает историю анализов в `localStorage`.
- `src/motion.js` — добавляет аккуратные анимации появления секций, карточек и графиков через IntersectionObserver.
- `src/navigation.js` — управляет активным пунктом навигации в sidebar и плавным переходом к разделам.
- `src/sidebarResize.js` — отвечает за изменение ширины sidebar и сохранение выбранной ширины.
- `src/theme.js` — переключает светлую и темную тему и сохраняет выбор пользователя.

## Текущие ограничения

- Направление метрик пока не настраивается: положительная динамика считается ростом, отрицательная — падением.
- Таблица результатов ограничивает отображение первыми 300 строками, чтобы интерфейс оставался отзывчивым.
- История хранит только результаты и метаданные, а не исходные файлы.
- Нет сохранения шаблонов сопоставления колонок.
- Нет отдельного режима для очень больших файлов с виртуализацией таблиц.
- Ручной режим сравнения в первой версии работает как список выбранных пар, без расширенного конструктора сложных сценариев.
- Нет backend и базы данных по архитектурному решению MVP.
