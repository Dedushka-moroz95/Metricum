# Ручной smoke-test Metricum

Этот чеклист нужен для быстрой проверки основных пользовательских сценариев после изменений в интерфейсе или логике приложения.

Smoke-test не требует npm, backend или установки зависимостей. Откройте `index.html` в браузере и используйте тестовые файлы из папки `tests/fixtures/`.

## Тестовые файлы

- `tests/fixtures/period-jan.csv` — отдельный файл за январь.
- `tests/fixtures/period-feb.csv` — отдельный файл за февраль.
- `tests/fixtures/period-mar.csv` — отдельный файл за март.
- `tests/fixtures/single-file-periods.csv` — один файл с колонкой периода `Month`.

Во всех файлах есть общий объект сравнения `Name`, подразделение `Team` и показатели:

- `Sales`
- `Quality %`
- `Errors`
- `AHT`

## 1. Несколько файлов

1. Откройте `index.html`.
2. В режиме источника данных выберите `Несколько файлов`.
3. Загрузите:
   - `period-jan.csv` в `Период 1`;
   - `period-feb.csv` в `Период 2`.
4. В блоке сопоставления выберите объект сравнения `Name` для обоих периодов.
5. Добавьте показатели `Sales`, `Quality %`, `Errors`.
6. Нажмите `Сравнить`.
7. Проверьте, что:
   - появился Dashboard;
   - появилась таблица результатов;
   - в результатах есть `Anna Kuznetsova`, `Boris Petrov`, `Elena Smirnova`;
   - `Maria Ivanova` отображается как объект, которого не было в январе;
   - процентные показатели отображаются как проценты.

## 2. Один файл с периодами

1. В режиме источника данных выберите `Один файл`.
2. Загрузите `single-file-periods.csv`.
3. В качестве колонки периода выберите `Month`.
4. В качестве объекта сравнения выберите `Name`.
5. Добавьте показатели `Sales`, `Quality %`, `Errors`.
6. Нажмите `Сравнить`.
7. Проверьте, что:
   - приложение собрало периоды `January`, `February`, `March`;
   - Dashboard и таблица результатов построились без повторной загрузки файлов;
   - Preview таблиц показывает данные по виртуальным периодам.

## 3. Ручные пары сравнения

1. Используйте режим `Один файл` и файл `single-file-periods.csv`.
2. Выберите ручной формат сравнения.
3. Добавьте пары:
   - `February - January`;
   - `March - February`.
4. Нажмите `Сравнить`.
5. Проверьте, что:
   - в таблице результатов видны выбранные пары;
   - график показывает динамику по выбранному показателю;
   - подписи периодов соответствуют ручным парам.

## 4. Фильтры

1. После расчета откройте блок фильтров в sidebar.
2. Введите в поиск сотрудника `Anna`.
3. Нажмите `Отфильтровать`.
4. Проверьте, что Dashboard и таблица обновились только после нажатия кнопки.
5. Очистите поиск, выберите фильтр только по улучшениям и нажмите `Отфильтровать`.
6. Проверьте, что остались только положительные изменения.
7. Нажмите `Сбросить фильтры`.
8. Проверьте, что исходные результаты вернулись.

## 5. История анализов

1. После успешного сравнения нажмите `Сохранить анализ`.
2. Введите название `Smoke-test`.
3. Сохраните анализ.
4. Обновите страницу.
5. Проверьте, что запись осталась в истории.
6. Откройте сохраненный анализ.
7. Проверьте, что восстановились:
   - Dashboard;
   - таблица результатов;
   - выбранные показатели;
   - графики.
8. Проверьте закрепление, переименование и удаление записи.

## 6. CSV экспорт

1. После расчета нажмите экспорт в CSV.
2. Проверьте, что файл скачался.
3. Откройте CSV и убедитесь, что в нем есть строки сравнения и рассчитанная динамика.

## 7. Excel экспорт

1. После расчета нажмите экспорт в Excel.
2. Откройте `.xlsx` файл.
3. Проверьте, что в книге есть русские вкладки:
   - `Обзор`;
   - `Сводка`;
   - `Сравнение`;
   - `Графики`;
   - `Отсутствующие`;
   - `Дубли`.
4. Проверьте, что на вкладке `Сравнение` есть данные сравнения.
5. Проверьте, что графики выгружены в виде PNG и не перекрывают таблицы.
6. Для показателя `Quality %` проверьте, что процентные столбцы отображаются в процентном формате.

## 8. Быстрые регрессионные проверки

1. Переключите светлую и темную тему.
2. Проверьте, что подсказки FAQ не перекрываются таблицами.
3. Проверьте, что tooltip графика появляется на нужной полосе.
4. Проверьте, что resize sidebar работает.
5. Проверьте, что в консоли браузера нет ошибок.
